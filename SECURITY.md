# Security Policy

## Supported versions

Security fixes are currently provided for the latest state of the `main` branch that powers the live site.

| Version | Supported |
| ------- | --------- |
| `main`  | Yes |
| older snapshots | No |

## Reporting a vulnerability

Please do **not** open a public GitHub issue for security vulnerabilities.

Instead, report security concerns privately to:

- `dan@newlocalmedia.com`

Please include:

- a clear description of the issue
- affected file(s), page(s), or workflow(s)
- reproduction steps or proof of concept if available
- impact assessment if known

## Response targets

- Initial acknowledgement: within 5 business days
- Status update after review: within 10 business days when feasible

## Scope

This repository is a static GitHub Pages site and related build scripts. Relevant reports may include:

- script or automation issues that could expose secrets or unsafe behavior
- client-side issues with meaningful security impact
- supply-chain or dependency concerns affecting this repository
