Write About Security Without FUD

For open source software providers, this editorial reference answers the question, ”How should I write about security?” It sets voice, tone, terminology, and technical formatting rules for people communicating about risk and vulnerability, with a special emphasis on the WordPress ecosystem. Build trust. Reduce fear, uncertainty, and doubt (FUD).

Make your security writing clear, direct, and empowering — especially when you’re explaining vulnerabilities, risk, remediation, and why our trust in open source is well-founded.
