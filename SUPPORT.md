# Support

## How to get help

Use the option that best matches your need:

### Bug reports

Open a GitHub issue using the bug report template when something on the site is broken or incorrect.

### Feature requests

Open a GitHub issue using the feature request template for improvements to:

- layout
- content presentation
- metadata
- automation
- repository curation

### Content corrections

If a repository summary, link, label, or generated project page is inaccurate, open an issue with the relevant URL and the proposed correction.

### Security issues

Do **not** use public issues for security concerns.

See [SECURITY.md](SECURITY.md).

## Response expectations

This is a maintained but lightweight public-facing repository. Response times may vary, but clear, specific reports are much easier to act on quickly.
